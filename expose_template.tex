\section{Introduction and Motivation}

Industrial production systems are characterized by high complexity due to fluctuating workloads, variable processing times and limited machine availability. Efficient production planning is therefore a key challenge in manufacturing environments. In particular, flexible job-shop scheduling requires assigning jobs to machines while respecting resource constraints.

Traditional approaches rely on predefined heuristics or rule-based strategies (e.g., SPT or EDD)~\cite[Chapter 3]{leungHandbookSchedulingAlgorithms2004}. While effective in stable environments, they lack adaptability to dynamic changes such as machine failures or new job arrivals.

Advances in artificial intelligence have introduced new approaches for solving complex decision-making problems. Reinforcement learning (RL) has shown promising results for dynamic scheduling~\cite{waschneckOptimizationGlobalProduction2023, waschneckDeepReinforcementLearning2018, zhangDeepReinforcementLearning2023}, but typically requires extensive training and retraining when system conditions change.

Large language models (LLMs) have recently demonstrated strong capabilities in reasoning, planning and tool interaction~\cite{yaoREACSYNERGIZINGREASONING2023, schickToolformerLanguageModels}. When used as agents, they can process structured inputs and generate decisions, making them a potential alternative for flexible production control~\cite{vyasAutonomousIndustrialControl2024a, xiaControlIndustrialAutomation2025a}.

This work evaluates LLM-based agents for production scheduling in flexible job-shop environments, focusing on their ability to generate valid and efficient decisions under varying system conditions.



\section{State of the Art}

Classical scheduling approaches include rule-based heuristics and optimization methods aimed at minimizing metrics such as makespan or waiting time~\cite[Chapter 3]{leungHandbookSchedulingAlgorithms2004}.

More recently, reinforcement learning has been successfully applied to dynamic scheduling problems, demonstrating the ability to learn adaptive strategies in complex environments~\cite{zhangDeepReinforcementLearning2023, waschneckDeepReinforcementLearning2018, waschneckOptimizationGlobalProduction2023}.


At the same time, current research explores the use of large language models as agents that are capable of reasoning and interacting with external systems. Foundational work shows that large language models can combine reasoning and action capabilities and integrate external tools into decision-making processes. Yao et al.~\cite{yaoREACSYNERGIZINGREASONING2023}, for example, demonstrate that language models can structure and solve complex tasks by combining reasoning and acting. Schick et al.~\cite{schickToolformerLanguageModels} further show that language models can learn to use external tools autonomously in order to extend their capabilities.

Initial work applies LLM agents to industrial control systems~\cite{xiaControlIndustrialAutomation2025a, vyasAutonomousIndustrialControl2024a}, highlighting their potential as flexible decision-making components.

However, their application to production scheduling remains largely unexplored, particularly with respect to systematic evaluation of performance, robustness and decision quality.



\section{Problem Definition and Research Questions}

This work evaluates the decision-making capabilities of LLM-based agents in flexible job-shop environments. The following research questions guide the investigation:

\begin{itemize}

\item \textbf{Performance and Real-Time Suitability}
\begin{itemize}
    \item To what extent can LLM-based agents generate valid and effective scheduling decisions?
    \item How suitable are LLM-based agents for real-time scheduling in terms of response time and latency?
\end{itemize}

\item \textbf{Types of Scheduling Decisions}
\begin{itemize}
    \item Which types of scheduling decisions (e.g., job selection, machine assignment, sequencing) can be reliably performed?
\end{itemize}

\item \textbf{Factors Influencing Decision Quality}
\begin{itemize}
    \item How does input representation (JSON, tabular, text) affect decision quality?
    \item How does increasing system complexity affect performance?
\end{itemize}

\item \textbf{Prompt Design}
\begin{itemize}
    \item How does prompt design influence decision quality and consistency?
\end{itemize}

\item \textbf{Comparison with Classical Methods (optional)}
\begin{itemize}
    \item (Optional) How does the performance of LLM-based scheduling approaches compare to classical heuristic methods?
\end{itemize}

\end{itemize}



\section{Planned Methodology}

This work combines a literature review with an experimental evaluation in a simulated production environment.

A flexible job-shop simulation models a simplified production system with multiple machines, different job types and dynamic job arrivals. A LLM-based agent observes the current system state and generates scheduling decisions, such as job selection and machine assignment.

The experiments vary input representation (JSON, tabular, text), prompt design and system complexity (e.g., number of machines, jobs and dynamic disruptions) in order to examine their influence on decision quality. Multiple runs per configuration are conducted to analyze stability and reproducibility.

Performance is evaluated using standard scheduling metrics such as makespan, flow time and machine utilization. Additionally, the validity and consistency of the generated decisions are analyzed, particularly with respect to compliance with system constraints and the reproducibility of decisions under identical states.

(Optional) The performance of the LLM-based scheduler is compared with classical heuristic scheduling strategies such as First Come First Serve (FCFS), Shortest Processing Time (SPT) and Earliest Due Date (EDD).

\subsection{Data}

The experiments use simulated production data based on a flexible job-shop environment. The system is modeled as a network of machines and connected conveyor belts.

Synthetic datasets simulate dynamic conditions, including varying job arrival rates, stochastic processing times, machine availability and possible disruptions such as machine failures.


The simulation environment provides the scheduling agent with the current system state, including information about machine states, waiting jobs, processing times, queues and transport times. This production state is provided to the LLM in different representations, for example as JSON structures, tabular formats, or natural language descriptions.

\subsection{Planned Model / Experiment / Analysis}

The experimental setup consists of a simulated flexible job-shop environment in which scheduling decisions must be made dynamically.

The LLM acts as a decision-making component that generates scheduling actions based on the current system state. These actions include job selection, assignment to a machine and the determination of processing order. The generated actions are then checked for validity and executed in the simulation system.

Production planning comprises several decision types, including job selection, machine assignment and integrated scheduling decisions, which are systematically investigated in the experiments.

The experiments systematically vary the input representation (e.g., JSON, tabular, textual), the prompt design and the complexity of the production environment (e.g., number of machines, jobs and disruptions). In addition, multiple runs are carried out per scenario to ensure the stability and reproducibility of the results.

The analysis focuses on the quantitative evaluation of scheduling performance as well as on the investigation of robustness, decision validity and sensitivity to different input representations and system configurations.

\subsection{Planned Evaluation}

The evaluation is based on common performance metrics from the production scheduling literature.

Possible evaluation metrics include:

\begin{itemize}
\item makespan (total completion time)
\item average flow time of jobs
\item machine utilization
\item job waiting time
\item number of tardy jobs
\end{itemize}

In addition to the quantitative metrics, LLM-specific evaluation criteria are considered. These include in particular the validity of the generated decisions (compliance with system constraints), the consistency of decisions under identical inputs and the sensitivity to different input representations and prompt variants.

In addition, the decision time of the LLM is analyzed to assess its suitability for real-time applications.
