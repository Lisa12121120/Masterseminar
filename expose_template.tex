\section{Introduction and Motivation}

Industrial production systems are characterized by high complexity due to fluctuating workloads, variable processing times and limited machine availability. Efficient production planning is therefore a key challenge in manufacturing environments. In particular, flexible job-shop scheduling requires assigning jobs to machines while respecting resource constraints.

Traditional approaches rely on predefined heuristics or rule-based strategies (e.g., SPT or EDD)~\cite[Chapter 3]{leungHandbookSchedulingAlgorithms2004}. While effective in stable environments, they lack adaptability to dynamic changes such as machine failures or new job arrivals.

Advances in artificial intelligence have introduced new approaches for solving complex decision-making problems. Reinforcement learning (RL) has shown promising results for dynamic scheduling~\cite{WASCHNECK20181264, waschneckDeepReinforcementLearning2018, zhangDeepReinforcementLearning2023}, but typically requires extensive training and retraining when system conditions change.

Large language models (LLMs) have recently demonstrated strong capabilities in reasoning, planning and tool interaction~\cite{yaoREACSYNERGIZINGREASONING2022, schickToolformerLanguageModels2023}. When used as agents, they can process structured inputs and generate decisions, making them a potential alternative for flexible production control~\cite{vyasAutonomousIndustrialControl2024a, xiaControlIndustrialAutomation2025a}.

This work evaluates LLM-based agents for production scheduling in flexible job-shop environments, focusing on their ability to generate valid and efficient decisions under varying system conditions.



\section{State of the Art}

Classical scheduling approaches include rule-based heuristics and optimization methods aimed at minimizing metrics such as makespan or waiting time~\cite[Chapter 3]{leungHandbookSchedulingAlgorithms2004}.

More recently, reinforcement learning has been successfully applied to dynamic scheduling problems, demonstrating the ability to learn adaptive strategies in complex environments~\cite{zhangDeepReinforcementLearning2023, WASCHNECK20181264}.


At the same time, current research explores the use of large language models as agents that are capable of reasoning and interacting with external systems. Foundational work shows that large language models can combine reasoning and action capabilities and integrate external tools into decision-making processes. Yao et al.~\cite{yaoREACSYNERGIZINGREASONING2022}, for example, demonstrate that language models can structure and solve complex tasks by combining reasoning and acting. Schick et al.~\cite{schickToolformerLanguageModels2023} further show that language models can learn to use external tools autonomously in order to extend their capabilities.

Initial work applies LLM agents to industrial control systems~\cite{xiaControlIndustrialAutomation2025a, vyasAutonomousIndustrialControl2024a}, highlighting their potential as flexible decision-making components.

However, their application to production scheduling remains largely unexplored, particularly with respect to systematic evaluation of performance, robustness and decision quality.


\section{Problem Definition and Research Questions}

This work evaluates the decision-making capabilities of LLM-based agents in flexible job-shop environments. The following research questions guide the investigation:

\begin{itemize}

\item \textbf{Performance and Real-Time Suitability}
\begin{itemize}
    \item To what extent can LLM-based agents generate valid and effective scheduling decisions?
    \item How suitable are LLM-based agents for real-time scheduling in terms of response time and latency?
\end{itemize}

\item \textbf{Types of Scheduling Decisions}
\begin{itemize}
    \item Which types of scheduling decisions (e.g., job selection, machine assignment, sequencing) can be reliably performed?
\end{itemize}

\item \textbf{Factors Influencing Decision Quality}
\begin{itemize}
    \item How does input representation (JSON, text) affect decision quality?
    \item How does system dynamics (static vs. dynamic environments) affect performance?
\end{itemize}

\item \textbf{Prompt Design}
\begin{itemize}
    \item How does prompt design influence decision quality and consistency? (prompt design: basic vs. chain-of-thought prompting)
\end{itemize}

\item \textbf{Comparison with Classical Methods (optional)}
\begin{itemize}
    \item (Optional) How does the performance of LLM-based scheduling approaches compare to classical heuristic methods?
\end{itemize}

\end{itemize}



\section{Planned Methodology}

This work combines a literature review with an experimental evaluation in a simulated production environment.

A simplified flexible job-shop simulation is developed to model a production system consisting of a fixed number of machines and jobs with sequential operations. The simulation abstracts conveyor systems and focuses on machine allocation and job sequencing decisions. Some operations can be processed on multiple alternative machines, while others require a specific machine. Multiple jobs can be present in the system simultaneously.

The LLM-based agent observes the current system state and generates scheduling actions. The output of the LLM is structured as a machine-readable set of job-machine assignments. The generated decisions are validated and executed within the simulation environment.

Two different system scenarios are considered:

\begin{itemize}
    \item \textbf{Static scenario:} A fixed set of jobs is processed without changes during execution.
    \item \textbf{Dynamic scenario:} New jobs arrive during execution and machines may become temporarily unavailable due to failures or become available again after repairs.
\end{itemize}

The experiments focus on three key factors influencing LLM performance:

\begin{itemize}
    \item \textbf{Input representation:} JSON vs. natural language descriptions
    \item \textbf{Prompt design:} basic instructions vs. chain-of-thought prompting
    \item \textbf{System dynamics:} static vs. dynamic environment
\end{itemize}

To ensure comparability, the performance of the LLM-based scheduler is evaluated against a classical heuristic baseline, specifically the Earliest Due Date (EDD) rule.

Multiple runs per configuration are conducted to analyze stability and reproducibility.

\subsection{Data}

Synthetic job data is generated to simulate realistic production scenarios. Each job consists of a sequence of operations that must be processed in a predefined order and has an associated due date.

Processing times depend on the job type. In the dynamic scenario, additional variability is introduced through random job arrivals and machine failures.

The system state includes information about:

\begin{itemize}
    \item available machines and their status (idle, busy, unavailable, and expected finish times) 
    \item jobs waiting for processing
    \item remaining operations per job
    \item due dates
\end{itemize}



This state is provided to the LLM in different representations, namely structured JSON format and natural language descriptions.

\subsection{Planned Model / Experiment / Analysis}

The experimental setup consists of a simulation environment in which scheduling decisions must be made sequentially.

At each decision step, the LLM receives the current system state and generates a set of scheduling actions, each consisting of selecting a job and assigning it to a machine. Each generated action is validated individually. Valid and conflict-free actions are executed, while invalid or conflicting actions are discarded and recorded for evaluation. To reduce complexity, the model abstracts transport processes by assigning fixed transfer times between machines instead of explicitly modeling routing decisions. Multiple jobs can be processed in parallel, depending on machine availability.


The experiments follow a factorial design with the following factors:

\begin{itemize}
    \item Scenario: static vs. dynamic
    \item Input representation: JSON vs. text
    \item Prompt design: basic vs. chain-of-thought
\end{itemize}

This results in a total of 8 experimental configurations. Each configuration is repeated multiple times to ensure robustness of the results.

The analysis focuses on comparing the performance of the LLM across different configurations and against the heuristic baseline. In addition to quantitative metrics, qualitative analysis is conducted to identify typical error patterns and limitations of the LLM-based approach.

\subsection{Planned Evaluation}

The evaluation is based on standard performance metrics from production scheduling literature.

\textbf{Efficiency metrics:}
\begin{itemize}
    \item makespan (total completion time)
    \item average flow time
\end{itemize}

\textbf{Due date related metrics:}
\begin{itemize}
    \item number of tardy jobs
\end{itemize}

\textbf{Resource utilization:}
\begin{itemize}
    \item machine utilization
\end{itemize}

\textbf{LLM-specific metrics:}
\begin{itemize}
    \item invalid decision rate (violations of system constraints)
    \item response time (latency per decision)
    \item rate of outputs with invalid format
    \item rate of invalid assignments (e.g., non-existing machines or jobs)
    \item rate of conflicting assignments (e.g., multiple jobs assigned to the same machine)
\end{itemize}

In addition to quantitative evaluation, the robustness and consistency of the LLM are analyzed by examining repeated runs under identical conditions.

Furthermore, a comparison with the EDD heuristic provides a baseline for assessing the effectiveness of the LLM-based approach.
